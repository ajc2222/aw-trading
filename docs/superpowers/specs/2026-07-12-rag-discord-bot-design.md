# RAG Discord Bot — Design Spec
*Date: 2026-07-12*

## Overview

A local Python application that ingests YouTube trading content from four creators, stores it in a ChromaDB vector database, and exposes it via a Discord bot with per-creator slash commands. Answers are generated by Gemini 2.0 Flash (free tier) using retrieved transcript chunks as context.

---

## Architecture

```
YouTube Channels
      │
      ▼
 ingest.py  (run manually or on a schedule)
  ├── youtube-transcript-api  →  pulls transcripts by channel/playlist
  ├── chunks text into ~500-token segments with 50-token overlap
  └── embeds + upserts into ChromaDB  (metadata: source, video_id, title, url)

ChromaDB  (local persistent store, single collection)
      │
      ▼
 bot.py  (always-on Discord process)
  ├── /ask-fluxtrades  <question>
  ├── /ask-aidenomics  <question>
  ├── /ask-bionicnq    <question>
  └── /ask-gxt         <question>
        │
        ▼  filters collection by source=<creator>
        retrieves top-5 chunks
        builds prompt → Gemini 2.0 Flash API
        returns answer + source video titles as citations
```

Runs entirely on local machine. Two entry points: `ingest.py` (manual) and `bot.py` (continuous).

## Ingestion Pipeline (`ingest.py`)

- Input: playlist URLs per creator defined in `config.py`
- Uses `yt-dlp` for video metadata (title, URL, video ID)
- Uses `youtube-transcript-api` to fetch transcripts
- Chunks transcripts into ~500-token segments with 50-token overlap
- Embeds with Gemini `text-embedding-004` (free)
- Upserts into ChromaDB with metadata: `source`, `video_id`, `title`, `url`
- Deduplication: skips videos already present in the store by `video_id`
- CLI flag `--source <creator_key>` to re-index a single creator

## Discord Bot (`bot.py`)

### Commands

| Command | Creator |
|---|---|
| `/ask-fluxtrades <question>` | Flux Trades |
| `/ask-aidenomics <question>` | Aidenomics |
| `/ask-bionicnq <question>` | BionicNQ |
| `/ask-gxt <question>` | GXT |

### Request Flow

1. Filter ChromaDB collection by `source=<creator_key>`
2. Retrieve top-5 chunks most relevant to the question
3. Build prompt: system message establishes creator persona + instructs to answer only from context
4. Send to Gemini 2.0 Flash, get response
5. Reply in Discord with answer + "Sources" footer (video title + URL)
6. Split responses exceeding Discord's 2000-char limit

## What's Out of Scope

- Conversational memory (stateless Q&A only)
- Web dashboard or admin UI
- Automatic scheduled ingestion (manual `python ingest.py` for now)
- Hosting/deployment (local machine)
